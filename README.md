# LeetCode 推荐系统

## 项目概述

LeetCode 推荐系统是一个个性化算法学习助手，能够基于用户的学习进度和表现智能推荐适合的编程题目。本系统旨在帮助用户系统性地学习各种算法和数据结构，提供代码评估和反馈功能，让算法学习过程更高效、更有针对性。

## 主要功能

- **智能题目推荐**：根据用户学习历史和表现自动推荐最适合的下一题
- **算法类别全覆盖**：确保用户逐步接触所有核心算法类别，打下扎实基础
- **代码智能评估**：通过 GPT-4o 模型评估提交的代码，给予专业反馈
- **学习进度分析**：追踪和可视化用户在各算法类别的优势和不足
- **本地化解决方案**：完全在本地运行，所有数据保存在本地文件中

## 安装与使用

### 前置条件

- Python 3.8+
- OpenAI API 密钥（用于代码评估功能）

### 安装步骤

1. 克隆本仓库：
   ```
   git clone https://github.com/yourusername/leetcode-recommendation-system.git
   cd leetcode-recommendation-system
   ```

2. 安装依赖：
   ```
   pip install openai
   ```

3. 配置 OpenAI API 密钥：
   - 创建一个 `.env` 文件（基于 `.env.example`）

     ```
     OPENAI_API_KEY=your_openai_api_key_here
     ```
   - 或者，设置环境变量：
     ```bash
     # Linux/Mac
     export OPENAI_API_KEY="your-api-key-here"
     
     # Windows
     set OPENAI_API_KEY=your-api-key-here
     ```

### 使用方法

1. 运行系统：
   ```
   python recommendation_system.py
   ```

2. 系统功能菜单：
   - 获取推荐题目
   - 查看弱势领域

   - 提交当前题目解答

## 推荐系统实现细节

### 推荐阶段与策略

系统使用三阶段推荐策略，根据用户进度智能调整：

1. **初始阶段（首次使用）**
   - 从不同算法类别中随机选择 Easy 难度题目
   - 确保类别多样性，建立基础知识面

2. **类别探索阶段（尚未尝试所有类别）**
   - 优先推荐 14 个核心算法类别中未尝试过的类别
   - 针对新类别优先推荐 Easy 难度题目，其次是 Medium 难度
   - 目标：确保用户接触所有基础算法类别，建立全面知识体系

3. **性能导向阶段（已尝试所有类别）**
   - 基于用户在各类别的表现计算每个题目的推荐分数
   - 优先推荐用户表现较弱类别的题目，鼓励加强薄弱环节
   - 根据用户能力水平动态调整推荐题目难度
   - 总尝试次数<14时，不推荐 Hard 难度题目

### 推荐分数计算逻辑

每个题目的推荐分数基于以下因素：

- **基础分数**：Easy (3.0) > Medium (2.0) > Hard (0.5)
- **类别表现调整**：成功率低的类别获得更高权重
- **难度匹配**：根据用户在该类别的成功率调整不同难度题目的权重
- **重复处理**：已解决题目大幅降低分数，已尝试未解决题目略微提高分数

这种多层次推荐系统确保用户学习路径既全面又个性化，帮助用户系统地掌握所有算法类别。

## 技术实现

### 技术栈

- **Python**：核心实现语言
- **JSON**：数据存储格式
- **OpenAI API**：代码评估

### 核心数据结构

- **问题树 (Problem Tree)**：
  - 树形结构，用于组织和分类所有题目
  - 根节点连接三个难度节点（Easy, Medium, Hard）
  - 每个难度节点下包含该难度的所有问题节点
  - 支持O(1)时间复杂度的按难度查询

- **哈希表/字典 (Hash Map/Dictionary)**：
  - 存储题目和用户进度信息
  - 维护类别到题目的映射、题目名称到解题历史的映射
  - 提供O(1)时间复杂度的查找操作
  - 在系统中通过Python字典实现

- **列表/数组 (List/Array)**：
  - 存储每个难度下的题目、推荐题目列表
  - 支持按分数排序的推荐结果
  - 用于存储用户解题历史
  - 系统中大量使用Python列表

- **枚举类型 (Enum)**：
  - 用于难度等级的定义（Easy, Medium, Hard）
  - 提供类型安全和代码可读性
  - 通过Python的enum模块实现

- **自定义类 (Custom Classes)**：
  - ProblemNode：表示单个题目，包含名称、难度、类别、链接和分数
  - DifficultyNode：表示难度节点，包含该难度下的所有题目
  - RecommendationSystem：主类，处理推荐逻辑和用户交互

- **集合 (Set)**：
  - 用于存储已尝试过的类别、确保推荐类别的多样性
  - 通过Python集合实现，支持O(1)的成员检测

- **JSON数据格式**：
  - 用于数据序列化和持久化存储
  - 保存题目库(problems.json)和用户进度(user_progress.json)
  - 程序运行时转换为Python的字典和列表等数据结构

### 核心算法

- **推荐评分算法**：
  - 根据多维度因素计算每个题目的推荐分数
  - 考虑用户在各类别的表现、题目难度和之前尝试情况
  - 算法根据学习阶段动态调整推荐策略

- **排序和选择算法**：
  - 使用Python内置排序(sort)，对问题按推荐分数排序
  - 时间复杂度为O(n log n)
  - 从排序后的列表中选择最适合的题目

- **多阶段推荐策略**：
  - 初始阶段：推荐多样化类别的题目
  - 类别探索阶段：优先推荐未尝试的类别
  - 性能导向阶段：根据用户表现智能推荐题目

- **GPT评估集成**：
  - 通过OpenAI API发送代码进行评估
  - 使用正则表达式(regex)解析评估结果
  - 提取分数和反馈信息

- **进度追踪算法**：
  - 记录和分析用户在各算法类别的表现
  - 计算成功率和尝试次数
  - 识别用户的强项和弱项

## 系统工作流程

1. **启动系统**：读取题目库和用户进度，构建问题树
2. **获取推荐**：系统分析用户进度，推荐适合的题目
3. **选择题目**：用户从推荐列表中选择一道题目
4. **解决题目**：系统生成题目模板，用户在模板中编写解决方案
5. **提交评估**：用户提交解决方案，系统调用 GPT-4o 评估代码
6. **进度更新**：系统记录评估结果，更新用户进度
